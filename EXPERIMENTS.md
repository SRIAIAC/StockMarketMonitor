## 2026-07-10 (2)

### New feature — OrchestratorAgent: LLM market briefing + rule-decided anomaly-triggered re-runs
- **Ask:** "build an orchestrating agent that uses LLM to give market briefing basis all agents and detects anomalies and decides to run specific agents." Question first: this project had no meta-agent — `scheduler.py`'s `run_all_agents()` is a fixed sequential runner, not an agent, and `RecommendationAgent` reads a few other agents' tables but only through one fixed formula, not general cross-agent synthesis.
- **Design (kept consistent with `AlertAgent`'s existing "rules decide, AI only narrates" split — see `agents/README.md`):** the *decision* of which agents to re-trigger is entirely deterministic (`analysis/anomaly_rules.py`), never left to the LLM — same reasoning as why `rules.py` decides alert severity/`needs_ai` before `claude_client.py` is ever called. Claude's only job is turning already-decided structured facts into a readable paragraph. Concretely:
  - `analysis/anomaly_rules.py` — 5 threshold-based detectors (`Anomaly` dataclass): a >5% single-ticker price move (→ re-run News/Social/Alert), a ≥15-point RiskSnapshot jump vs. the previous snapshot (→ Risk's own downstream, Recommendation/Alert), market-wide news+social sentiment below -0.4 over the last hour with ≥5 samples (→ Social/Alert), a sector momentum_score ≥85 or ≤15 — i.e. genuinely extreme, not just trending (→ Recommendation), and ≥3 warning/critical alerts in 15 minutes (→ Recommendation/Risk). `merge_triggered_agents()` unions + dedupes across whichever fired.
  - `agents/orchestrator_agent.py` (`OrchestratorAgent`) — gathers a structured snapshot across `Price`, `NewsItem`/`SocialPost`, the last 2 `RiskSnapshot` rows, live sector momentum, recent `Alert` count, and the top `Recommendation` pick; runs the 5 detectors; instantiates and `run_safe()`s just the specific agent classes an anomaly flagged (not a full `run_all_agents()` sweep); then builds a plain-text context block and calls the new `claude_client.generate_briefing()`.
  - `claude_client.generate_briefing(context, has_anomalies)` — new function, same conventions as the existing `triage_and_explain`/`recommend_reason`: Haiku by default, escalates to Sonnet only when `has_anomalies=True` (mirrors the existing Haiku→Sonnet gate, just keyed off the rule-based anomaly count instead of a single alert's `HIGH_IMPACT` flag), sha256-prompt-cached, never raises. Without an `ANTHROPIC_API_KEY` (the default in this environment), `OrchestratorAgent` falls back to a deterministic templated headline/summary built from the same structured facts — never a blank panel.
  - New `MarketBriefing` model (append-only, one row per run) + `GET /api/briefing` (`routes_agents.py`) returning `{headline, summary, anomalies[], agents_triggered[], ai_generated, computed_at}`.
  - Scheduling: its own **15-minute interval** (independent of the other agents' 30-minute cadence — deliberately faster, so it can react *between* full sweeps, otherwise "detects anomalies and decides to run agents" would be redundant with the sweep that already runs everything anyway), plus once more at the tail of `run_all_agents()`.
- **Deliberately not done, flagged rather than silently decided:** `OrchestratorAgent` is **not** added as an 11th card to the Overview page's `AgentStatusRow` — that row was built to exactly match the product spec's "10 agents / 10-10 Active" (see the earlier 2026-07-10 entry), and Orchestrator is a meta-agent over those 10, not an 11th data source of its own. Its own liveness is instead surfaced inline on the new `MarketBriefingPanel` (`orchestrator_active`/`orchestrator_last_run` fields on `/api/briefing`) and the panel was placed at the very top of the Overview page, above the agent-status row, since a synthesized briefing is naturally the first thing worth reading.
- **Tested:** ran `OrchestratorAgent().run()` directly against the dev DB — real detection fired twice in the same run (CDSL +6.34% price anomaly, Realty sector at momentum 100/100) and correctly re-triggered exactly `news, social, alert, recommendation` (not the unrelated agents — market/econ/youtube/corporate_action/regulatory/risk were correctly left alone for a price-and-sector anomaly). Verified the deterministic fallback briefing text (no API key configured) reads correctly and is never blank. `npm run build` — zero TypeScript errors. Live backend+frontend, curl-verified `/api/briefing`'s shape and confirmed the raw HTTP response bytes are correct UTF-8 (`·`/`—` render fine — an earlier apparent "mojibake" was traced to `python -m json.tool` misreading piped stdin under this Windows console's codepage during my own diagnostic command, not a real encoding bug in the app; verified with a direct DB byte-level check and a hex dump of the actual response bytes). Headless-Chromium screenshot of the Overview page confirmed the `MarketBriefingPanel` renders correctly at the top: headline, summary, amber anomaly rows, and a "Re-triggered off-cycle" chip row listing the real agents that fired.
